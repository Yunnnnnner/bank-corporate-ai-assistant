# 银行公司业务知识库系统落地路径
## 三阶段·十二步实施方案

---

## 总体目标

构建覆盖**公司全业务线**的智能知识管理与辅助决策系统，实现：
1. 业务人员随时查询业务规则和操作规范
2. 审批人员快速调取合规依据和风控标准
3. 合规管理人员实时识别监管规则变化
4. 产品团队追踪竞品动态与市场机会

---

## 第一阶段：知识库底座建设（M1-M3）

### Step 1: 文档清单梳理与优先级排序

**P0（核心，必须首批入库）**
```
信贷产品类：
  □ 各类贷款产品说明书（流动资金/固定资产/并购/银团）
  □ 信贷业务操作规程（全流程版本）
  □ 授信审批权限管理办法
  □ 贷款定价管理办法
  □ 贷后管理操作规范

监管合规类：
  □ 固定资产贷款管理办法（2024版）
  □ 流动资金贷款管理办法（2024版）
  □ 商业银行大额风险暴露管理办法
  □ 并购贷款风险管理指引
  □ 银行业消费者权益保护管理办法
```

**P1（重要，第二批入库）**
```
  □ 票据业务操作手册
  □ 供应链金融管理办法
  □ 贸易融资操作规范
  □ 资产证券化管理规定
  □ 绿色信贷政策文件
```

**P2（补充，持续补充）**
```
  □ 各业务线FAQ文档
  □ 历史审批案例（脱敏）
  □ 监管问询答复口径
  □ 司法判例相关文件
```

### Step 2: 文档标准化处理

**格式统一化**
- 将 Word/PDF/Excel 统一转为结构化 Markdown
- 建立统一的文档元数据体系：

```json
{
  "doc_id": "SHA256哈希",
  "title": "文档名称",
  "category": "业务规则|监管文件|内部政策|法律法规",
  "product_line": "对公贷款|票据|ABS|供应链|贸易融资|投行",
  "document_type": "操作规范|产品手册|监管规定|内部制度",
  "effective_date": "生效日期",
  "expiry_date": "失效日期",
  "version": "版本号",
  "issuer": "发布机构",
  "keywords": ["关键词1", "关键词2"],
  "related_docs": ["关联文档ID"],
  "confidentiality": "公开|内部|机密"
}
```

**知识结构化提取**（对关键规则类文档）
- 自动提取"规则三元组"：条件 → 规则内容 → 例外
- 提取关键数字参数（利率、比例、期限等）建立参数数据库
- 建立知识图谱：产品→政策→监管法规的依赖关系

### Step 3: 分块策略优化（信贷专用）

**按业务文档特点定制分块规则**

| 文档类型 | 分块策略 | 分块大小 |
|---------|---------|---------|
| 法规/制度类 | 按条款分割（第X条） | 200-400字 |
| 产品手册类 | 按功能章节分割 | 400-600字 |
| 操作规程类 | 按流程步骤分割 | 300-500字 |
| FAQ类 | 一问一答作为最小单位 | 100-300字 |
| 利率/参数表 | 表格行为最小单位 | 50-150字 |

**特殊处理：数字参数提取**
```python
# 自动提取关键参数，建立独立索引
PARAMETER_PATTERNS = [
    r"不超过(\d+%)",          # 比例限制
    r"不低于(\d+%)",          # 最低要求
    r"不超过(\d+年|\d+个月)",  # 期限
    r"(\d+万元|\d+亿元)以上",  # 金额门槛
    r"(\d+个工作日)内",        # 时效要求
]
```

### Step 4: Embedding 优化

**中文信贷领域专用优化方案**
- 使用 `text2vec-large-chinese`（HuggingFace）作为本地 Embedding，专为中文优化
- 或使用智谱 Embedding-2 API（1024维，中文优化）
- 对关键专业术语建立同义词词典：
  ```
  流动资金贷款 ↔ 短贷 ↔ 周转贷款
  固定资产贷款 ↔ 中长期贷款 ↔ 项目贷款
  银行承兑汇票 ↔ 银承 ↔ BAB
  应收账款 ↔ 应收款 ↔ 债权
  ```

## 第二阶段：业务功能扩展（M4-M6）

### Step 5: 智能问答能力升级

**多场景问答适配**

| 使用场景 | 典型问题 | 系统回答要求 |
|---------|---------|------------|
| 业务咨询 | "小微企业申请信贷需要满足哪些条件？" | 完整列举条件，注明来源文件 |
| 操作查询 | "开具银行承兑汇票第几步需要上传发票？" | 精确步骤定位，引用操作手册 |
| 合规审查 | "这笔并购贷款比例是否超标？" | 给出判断依据和监管条款 |
| 参数查询 | "流动资金贷款最长期限是多少？" | 精确参数+政策依据+生效时间 |
| 比较分析 | "有追索保理和无追索保理有何区别？" | 结构化对比表格 |

**Chain of Thought（思维链）增强**
对于复杂合规判断，要求 LLM 分步推理：
```
系统提示词追加：
"对于合规性判断问题，请按以下步骤回答：
1. 识别问题涉及的业务类型
2. 找到相关监管规定
3. 提取关键参数/条件
4. 对照实际情况进行判断
5. 给出结论并注明依据"
```

### Step 6: 结构化知识库补充

**除了非结构化文档外，建立结构化规则数据库**

```sql
-- 产品规则表
CREATE TABLE product_rules (
    rule_id       TEXT PRIMARY KEY,
    product_code  TEXT,           -- 产品代码
    rule_type     TEXT,           -- 准入|额度|期限|利率|合规
    rule_name     TEXT,
    rule_value    TEXT,           -- 规则内容（可为JSON）
    min_value     NUMERIC,        -- 数值下限
    max_value     NUMERIC,        -- 数值上限
    unit          TEXT,           -- 单位（%/万元/年）
    condition     TEXT,           -- 适用条件
    exception     TEXT,           -- 例外情形
    source_doc    TEXT,           -- 依据文件
    effective_date DATE,
    expiry_date   DATE
);

-- 利率参数表
CREATE TABLE pricing_params (
    param_id      TEXT PRIMARY KEY,
    product_code  TEXT,
    customer_grade TEXT,          -- 评级
    guarantee_type TEXT,          -- 担保方式
    min_spread    NUMERIC,        -- 最低加点(BP)
    max_spread    NUMERIC,        -- 最高加点(BP)
    effective_date DATE
);
```

**混合检索增强**：结构化查询 + 向量检索双路召回

```python
def smart_retrieve(question):
    # 1. 判断是否为参数查询（精确匹配）
```

### Step 7: 监管变化追踪

**自动监控监管动态**
- 定时爬取：银保监/人行/证监会/银行间市场官网
- 关键词预警：涉及本行业务的新规、征求意见稿
- 自动触发：检测到相关新规 → 推送给合规人员 → 人工确认后更新知识库
- 版本管理：保留历史版本，明确新旧规则生效时间

## 第三阶段：智能化深化（M7-M12）

### Step 8: 智能审批辅助

**信贷条件预检功能**
用户输入贷款申请关键参数，系统自动检查是否满足准入条件：

```
输入：
  借款人：制造业企业，成立2年
  申请产品：流动资金贷款
  申请金额：3000万元
  担保方式：应收账款质押

系统输出：
  ✅ 经营年限：满足（成立满1年要求）
  ✅ 产品匹配：制造业符合流动资金贷款用途
  ⚠️  担保方式：应收账款质押率上限60%，质押充分率须>100%
  ⚠️  金额：超出支行审批权限（2000万），需报分行
  ✅ 行业：制造业无限制
  
  综合结论：可以受理，建议上报分行信贷审批，
  所需补充材料：[详细清单]
```

### Step 9: 合规智能检查

**贷款合同合规检查**

上传贷款合同 → 系统自动检查：
- 必备条款是否完整（借款金额、期限、利率、还款方式）
- 利率是否低于监管底线（LPR - 30BP）
- 受托支付条款是否按规定设置
- 担保条款是否符合规定

### Step 10: 多模态支持

**图片/表格文档处理**
- OCR提取扫描版PDF内容
- 表格识别（利率表、产品参数表）自动结构化
- 流程图识别 → 转换为文字描述

## 系统部署架构

### 轻量部署方案（单机/小团队）

```
┌────────────────────────────────────────┐
│          用户界面层                     │
│  CLI / Web UI (FastAPI + 简单前端)      │
├────────────────────────────────────────┤
│          应用层                         │
│  RAG Pipeline + Claude API              │
├────────────────────────────────────────┤
│          存储层                         │
│  ChromaDB (向量) + SQLite (元数据+FTS)  │
├────────────────────────────────────────┤
│          数据层                         │
│  本地文件系统 data/raw/                 │
└────────────────────────────────────────┘
硬件需求：4核 CPU + 16GB内存 + 100GB SSD
成本：零基础设施成本（本地运行）
```

### 企业级部署方案（生产环境）

```
┌──────────────────────────────────────────────┐
│                   负载均衡层                   │
│              Nginx / API Gateway               │
├──────────────┬───────────────────────────────┤
│   Web应用层  │         业务逻辑层              │
│  Vue/React   │   FastAPI + RAG Engine          │
├──────────────┴───────────────────────────────┤
│                    缓存层                       │
│              Redis (热点问答缓存)               │
├──────────────────────────────────────────────┤
│              分布式存储层                       │
│  Milvus (向量) | MySQL (元数据) | MinIO (文件)  │
├──────────────────────────────────────────────┤
│              基础设施层                         │
│       Docker / Kubernetes 容器化部署             │
└──────────────────────────────────────────────┘
硬件需求：4台 8核32GB服务器 + GPU服务器（可选）
部署方式：私有化部署（满足银行数据安全要求）
```
