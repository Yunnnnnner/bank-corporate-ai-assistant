# 供应链金融产品体系
## 参照建设银行供应链服务平台（CCB-SCF）

---

## 第一章 供应链金融逻辑框架

### 1.1 核心理念

供应链金融是以核心企业为中心，利用核心企业信用向其上下游供应商/经销商提供融资服务，打通供应链资金流动。

**关键特征**
- **自偿性**：融资与特定贸易交易绑定，还款来源明确
- **封闭性**：资金流、物流、信息流在封闭系统内流转
- **核心企业信用替代**：中小企业凭借与核心企业的贸易关系获得融资

### 1.2 三种模式的资金流向

```
【应付账款模式（上游）】
  供应商 ←── 建行融资（贴现/保理）
             ↑ 核心企业确认应付账款
  核心企业  ──→ 到期付款给建行

【买方融资模式（下游）】
  经销商 ←── 建行融资（买方信贷）
  核心企业 ──→ 担保/确认订单
  销售回款 → 建行还款

【存货融资模式】
  存货质押 ──→ 建行贷款
  货物销售回款 ──→ 还款
```

## 第二章 核心产品详解

### 2.1 e点通（应付账款融资）

**产品定义**
以核心企业对供应商的应付账款为基础，供应商凭借核心企业确认的应付账款向银行申请融资的业务。

**适用场景**
- 核心企业占用供应商账期（通常60-180天）
- 供应商需要提前变现，改善现金流
- 核心企业希望延长付款账期，同时帮助供应商降低融资成本

**操作流程**
```
Step 1: 核心企业与建行签署合作协议
Step 2: 供应商完成发货，取得发票
Step 3: 核心企业在建行平台确认应付账款（关键：账款真实性由核心企业背书）
Step 4: 供应商在线申请融资
Step 5: 建行审核（系统自动，T+0或T+1）
Step 6: 建行放款（扣除贴息后转账给供应商）
Step 7: 到期日，核心企业直接向建行还款
```

**定价逻辑**
```
融资利率 = 核心企业主体评级对应基准利率 + 少量加点
（通常低于供应商自身申请贷款利率，降低整体供应链融资成本）
```

**风控要点**
- 核心企业信用是第一道防线（核心企业越强，风险越低）
- 应付账款的真实性核查（防止虚假确权）
- 核心企业账期集中度管理
- 防止同一笔应付账款重复融资

---

### 2.2 e销通（下游买方融资）

**产品定义**
建设银行与供应链核心企业合作，借助核心企业信用增信，为核心企业推荐的下游客户（经销商/加盟商）提供全流程网上操作的网络融资服务。

**特点**
- 借款主体：经销商（下游）
- 信用基础：核心企业担保/推荐背书
- 还款来源：经销商销售回款（自偿）
- 操作模式：线上全流程，核心企业平台与建行系统对接

**适用场景**
- 经销商旺季提前备货，需要短期融资
- 经销商授信较弱，需借助核心企业信用
- 核心企业希望扩大销售网络，帮助经销商获得融资

### 2.3 保理业务

#### 2.3.1 国内保理

**有追索保理**
- 应收账款转让给银行（保理商）
- 债务人不付款时，银行向卖方追偿
- 卖方仍承担信用风险
- 融资成本较低

**无追索保理（真实出售）**
- 信用风险完全由银行承担
- 应收账款从卖方资产负债表出表
- 融资成本较高
- 需要对债务人信用进行深度评估

**保理业务准入矩阵**

| 债务人评级 | 账期 | 有追索 | 无追索 |
|---------|-----|-------|-------|
| AAA-AA | ≤90天 | 支持 | 支持（优先） |
| A | ≤90天 | 支持 | 需审批 |
| BBB | ≤60天 | 支持 | 一般不支持 |
| BB以下 | - | 不支持 | 不支持 |

**应收账款转让定价**
```
保理利率 = 债务人融资成本 + 保理综合费用
         = 债务人贷款LPR + 加点 + 手续费(0.1%-0.3%)
```

#### 2.3.2 国际保理

**双保理模式（最常用）**
```
出口商 ──应收账款转让──→ 出口保理商（建行）
                            │── 转让债权 ──→ 进口保理商（境外银行）
进口商 ──付款──────────→ 进口保理商
                            └── 转账 ──→ 出口保理商 ──→ 出口商
```

**优势**：利用进口地银行了解本地债务人信用，降低风险

## 第三章 订单融资

### 3.1 业务逻辑

订单融资是指以买方订单为基础，向卖方（供应商）提供采购原材料所需资金支持，形成订单→生产→发货→收款→还款的闭环。

**关键控制点**
- 订单真实性核查（防伪造订单）
- 物资采购合同、发票存档
- 货物完工后的发货确认
- 收款账户指定银行（封闭资金流）

### 3.2 与应付账款模式的区别

| 对比维度 | 应付账款融资 | 订单融资 |
|---------|-----------|---------|
| 时间节点 | 发货完成后（账款已形成） | 发货前（订单阶段） |
| 核心证据 | 已确认应付账款 | 买方订单 |
| 风险程度 | 较低（已交货） | 较高（生产风险+市场风险） |
| 适用场景 | 补充营运资金 | 扩大生产规模 |

## 第四章 供应链金融风险管理

### 4.1 系统性风险：核心企业风险

**警示指标**
- 核心企业公开评级下调
- 核心企业供应链付款账期延长（即时预警）
- 核心企业行业主管部门专项检查
- 媒体/舆情负面报道增加

**处置预案**
- 暂停新增核心企业项下业务
- 加速存量业务到期收回
- 核查应付账款确权真实性
- 提高保证金要求

### 4.2 操作风险：确权欺诈

**常见欺诈手段**
1. 虚构核心企业身份（冒充）
2. 核心企业内部人员勾结伪造确权
3. 同一笔应付账款向多家银行重复融资
4. 伪造采购合同和发票

**防控措施**
- 与核心企业对接系统级API（不依赖人工确认）
- 建立应收账款登记平台查询（中国人民银行征信中心）
- 合同、发票、物流三单一致验证
- 定期与核心企业财务对账

### 4.3 集中度管理

| 维度 | 集中度上限 |
|------|---------|
| 单一核心企业 | 供应链金融总敞口≤20% |
| 单一供应商 | ≤供应链金融总敞口5% |
| 单一行业 | ≤30% |
| 无追索保理总量 | ≤保理总量50% |
